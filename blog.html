There are many differences between the Local Storage and Session Storage. There are lots of similarities between these two storage. The main differences between these two are the duration of storing the Data.

Local Storage:

There are no expiration limit for Local Storage. Whenever a user visits as specific website they will get the access in the Local Storage data. Since we have to remove the local storage data in some occasion, we can do that by setting an expiration duration or deleting them manually. But the local storage data of private window or incognito tab will be deleted after the tab or window closed.


Session Storage:

For session storage, the expiration time is limited. Whenever a tab is created a session storage will be created and the storage will be different for the same website if they are opened in different tab. However, if we want to get access to the session storage of one tab from the other tab then we have to duplicate the tab. Closing the tab or window will automatically remove the session storage data.

Global Scope:

The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.


Block Scope:

Block scope is also a sub-type of local scope. The block scope can be defined as the scope of the variables inside the curly brackets {} . Now, these curly brackets can be of loops, or conditional statements, or something else. You are only allowed to refer to these variables from within the curly brackets {} .

The event loop is the secret behind JavaScript's asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.
All the codes stored in memory randomly and JS runtime engine run them one by one and store them in a place named HEAP. In this process when the engine face a function, promise or setTimeOut or something like that which will take some time to be executed, the engine will send them to a stack. And move to the next code. After finishing the task of the codes from stack they will be sent to a Queue. After the engine process all the code from the heap then the data from the Queue will be sent to to the heap to be executed by the engine. And this process is called the Event Loop in JS.
